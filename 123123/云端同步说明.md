# 全站数据同步开通步骤

当前系统已接入 **Supabase 云端同步**：谁增删改，其他人刷新/登录后都能看到。

## 你必须先做这一步（否则同步不会生效）

1. 打开 [Supabase Dashboard](https://supabase.com/dashboard) → 进入项目 `havxlphglhjgcfgwowae`
2. 左侧 **SQL Editor** → New query
3. 打开本目录的 `supabase_sync.sql`，全部复制粘贴后点 **Run**
4. 看到成功后，刷新网页（本地或 GitHub Pages 均可）
5. 右下角应出现「已同步云端数据 … 项」提示

## 同步范围

会自动同步（写入 localStorage 即上传云端）：

- 团队成员档案、账号权限、项目、任务、周报
- 通知/新闻/会议、文献/数据集/报告、共享文件**元数据**
- 标准/软著、系统配置、操作日志等

**不会同步：**

- API Key（本机密钥）
- 登录会话 / 登录失败次数
- 共享文件的**真实二进制内容**（仍在各浏览器 IndexedDB；元数据会同步。要真正共享文件内容需再接 Supabase Storage）

专利、论文本来就走独立表，继续按原逻辑同步。

## 使用说明

- 本地改完数据后约 0.3 秒自动上传
- 每 15 秒自动拉取一次云端
- 登录成功、切换模块时也会拉取
- 部署到 GitHub Pages 后，所有人共用同一云端，删除会同步

## 首次迁移建议

1. 先执行 SQL
2. 用管理员账号在**一份**最新数据的浏览器登录一次（会把本机数据上传云端）
3. 其他人强制刷新（Ctrl+F5）后再登录
