# 全站数据同步（已开通 · 全局联动）

现在**不需要再执行 SQL**。系统会自动把共享数据同步到云端（复用现有 Supabase `patents` 表中的同步占位记录），并在首页 / 门户 / 各业务模块之间保持同源。

## 怎么用

1. 强制刷新页面：`Ctrl + F5`
2. 登录前会先拉取云端账号与权限；登录后右下角会出现「已同步云端数据 … 项」
3. 任意人增删改后，其他人约 **5 秒（首页）/ 15 秒（其他页）** 内，或刷新 / 切换模块后即可看到
4. 同一浏览器多个标签页之间，通过 `storage` 事件即时触发同步

## 会同步什么（全局联动范围）

- **账号权限（重点）**：`accountData`、`permissionMatrix`、`passwordPolicy`、`loginLogData`
- **团队**：`teamMemberData`、`memberGradeYears`、`memberData`（成员档案改动会联动账号与任务/周报等归属字段）
- **项目**：纵向 / 横向 / 校级
- **协作**：任务、周报、请假申请、审批流、通知、新闻、会议
- **资源**：文献、数据集、报告、共享文件列表、文献对比数据
- **成果**：标准、软著、竞赛、**论文 `paperData`**、**专利台账 `patentMgmtData`（并镜像 `patentData`）**
- **门户公开内容**：`portalContentConfig_v1`、`portalFeedbackData_v1`
- **系统**：系统配置、操作日志、备份配置、指令模板等

论文 / 专利台账仍写入 Supabase 业务表；同时镜像到 KV，保证首页统计与公开门户与后台台账一致。

## 权限如何全局生效

1. 在「角色权限管理」勾选并点击「保存权限配置」→ 立即写入云端
2. 其他浏览器约 15 秒内自动拉取，侧边栏菜单与编辑按钮会按矩阵刷新
3. 「团队成员档案（编辑）」「账号管理（新建/删除）」等开关直接控制对应功能入口

## 不会同步什么

- API Key、登录会话（本机；会话会在账号库变更后自动重新对齐，避免刷新掉线）
- 过大的头像 base64（资料仍同步）
- 共享文件 / 数据集等**真实二进制**（只同步文件列表；文件本体走本机 IndexedDB 或网关 / MinIO）

## 专利台账

同步占位记录（`classification=__APP_SYNC__`）不会出现在专利列表里（已自动过滤）。  
业务专利会镜像到 `patentData`，供首页与门户展示。

## 验收建议

1. A 浏览器登录，新增论文 / 专利 / 门户文案
2. B 浏览器打开首页，等待 ≤15 秒，确认统计数字变化
3. 打开公开门户，确认成果数字与后台一致
4. 修改角色权限后，另一端侧边栏入口同步变化
