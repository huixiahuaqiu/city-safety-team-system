# 城市安全数智创新团队文献管理系统

## 📖 项目简介

城市安全数智创新团队文献管理系统是一个面向高校科研团队的综合性管理平台，集成文献管理、项目跟踪、成果展示、数据分析与 AI 智能工具于一体，帮助团队高效管理科研项目全生命周期。

## ✨ 核心功能

### 📂 团队管理

- **团队成员档案**：维护团队人员信息与发展历程
- **角色权限管理**：支持多层级权限分配
- **内部任务待办**：任务分配与进度跟踪
- **团队工作周报**：周报生成与归档

### 📋 项目管理

- **纵向项目**：国家级/省部级纵向课题管理
- **横向项目**：校企合作横向项目管理
- **校级项目**：校内科研项目管理

### 🏆 成果管理

- **专利台账**：专利申请、授权全流程跟踪
- **论文成果**：学术论文收录与统计
- **标准专著**：标准与专著管理
- **软著管理**：软件著作权登记管理

### 📚 资源中心

- **文献资料库**：支持 Excel 文献数据批量导入、检索与筛选
- **数据集资源库**：团队数据集统一管理
- **项目报告归档**：结题报告、中期报告归档
- **共享文件库**：团队内部文件共享

### 🤖 智能工具

- **模型训练管理**：训练任务调度与监控
- **AI 数据标注工具**：支持文本/图像数据标注
- **智能对话问答**：基于大模型的 AI 助手
- **文献对比分析**：多文献数据可视化对比
- **Excel 数据处理**：自然语言驱动的 Excel 智能分析
- **文档智能解析**：PDF/Word 文档自动解析

### 📊 成果可视化

- **核心数据看板**：多维度科研成果统计
- **年度对比分析**：历年成果数量与增长率对比
- **里程碑时间线**：重大成果可视化展示
- **分类成果展示**：按类别分类展示团队成果

### 🔧 系统管理

- **账号权限设置**：用户角色与权限管理
- **系统基础配置**：系统参数灵活配置
- **操作日志管理**：审计日志追踪
- **数据备份管理**：数据安全保障

## 🚀 快速开始

### 方式一：直接打开（基础功能）

双击 `index.html` 文件即可在浏览器中打开系统，可使用以下功能：

- 团队信息浏览
- 成果数据展示
- 本地数据管理（localStorage 存储）

### 方式二：完整功能（推荐）

#### 前置条件

- Node.js 16+（[下载安装](https://nodejs.org/)）
- Python 3.7+（可选，如需使用 Python 代理）

#### 1. 启动代理服务器

```bash
cd 123123
node server.cjs
```

启动成功后将看到如下输出：

```
Server running at http://localhost:3000
Homepage: http://localhost:3000/
```

#### 2. 访问系统

浏览器打开 `http://localhost:3000/` 即可使用完整功能。

### 方式三：Supabase 云端数据库

系统支持将数据存储到 Supabase 云端数据库，实现多端同步：

1. 在 [Supabase](https://supabase.com/) 创建项目
2. 创建所需数据表（patents、papers 等）
3. 启用 Row Level Security (RLS) 行级权限控制
4. 在 `index.html` 中配置 `SUPABASE_URL` 和 `SUPABASE_KEY`
5. 重启代理服务器

详细配置见下方 [配置说明](#配置说明) 章节。

## ⚙️ 配置说明

### 环境变量

系统通过界面输入框和配置文件管理敏感信息：

- **API Key**：阿里云百炼 API 密钥，在"OpenAI入口"模块中配置
- **Supabase URL**：Supabase 项目地址
- **Supabase Key**：Supabase anon/publishable 密钥

### 推荐配置方式

将敏感配置写入 `config.local.js` 文件（该文件不应提交到 Git），系统会自动读取：

```javascript
// config.local.js 示例
window.APP_CONFIG = {
    supabaseUrl: 'https://your-project.supabase.co',
    supabaseKey: 'your-publishable-key'
};
```

在 `index.html` 中确保在头部加载配置文件：

```html
<script src="config.local.js"></script>
```

### ⚠️ 安全注意事项

- **不要**在代码中硬编码 API 密钥
- **不要**将包含真实密钥的配置文件提交到 Git
- 建议在 `.gitignore` 中添加 `config.local.js`
- Supabase 务必启用 Row Level Security (RLS) 保护数据

## 💡 使用示例

## ❓ 常见问题

**Q: 为什么 AI 功能提示"调用失败"？**

A: 请检查：
1. 代理服务器是否正在运行（`node server.cjs`）
2. API Key 是否已正确配置
3. 网络连接是否正常

**Q: 代理服务器启动失败怎么办？**

A: 检查以下几点：
1. Node.js 是否已正确安装
2. 端口 3000 是否被其他程序占用
3. 防火墙是否阻止了该端口

**Q: 数据会丢失吗？**

A: 系统使用 localStorage 或 Supabase 云端数据库存储。localStorage 数据仅限本地浏览器，清除缓存会丢失；Supabase 数据存储在云端，持久保存。建议重要数据使用 Supabase。

**Q: 如何导出/导入数据？**

A: 各个管理模块均支持 Excel 格式的导入导出，点击对应模块的导入/导出按钮即可。

1. 在 `index.html` 中添加新的模块 div（`class="module"`）
2. 在侧边栏添加导航入口
3. 编写对应的 JavaScript 处理函数
4. 如需后端支持，在 `server.cjs` 中添加路由

### 修改主题

在 `index.html` 的 `<style>` 部分搜索以下变量进行全局样式修改：

- `--ach-primary`：主色调
- `--ach-bg`：页面背景
- `--ach-white`：卡片背景
