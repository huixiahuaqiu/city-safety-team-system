系统基础配置模块 全量精细化开发方案
（全局参数中心：一处修改，全系统生效，深度联动所有业务模块）
本模块是整个系统的全局参数中枢，所有业务模块（成员档案、任务待办、周报、备份、日志、上传、登录）均从这里读取配置，彻底替代代码硬编码，实现可视化修改、实时生效。方案包含完整分类配置项、数据库设计、缓存机制、全系统联动工具类、前后端完整代码。
一、模块整体设计
1.1 模块定位
系统级全局参数管理中心，统一维护所有基础配置项。管理员在页面修改参数后，全系统所有模块实时读取最新值，无需改代码、无需重启服务。
1.2 权限边界
可见 / 可修改：仅 超级管理员（导师）
读取权限：所有业务模块后端内部均可读取（通过工具类），普通用户无感知、无操作权限
1.3 核心设计原则
键值对存储：配置项动态扩展，新增配置不用改表结构
缓存优先：Redis 缓存高频读取，修改后自动刷新，性能无损耗
全链路联动：所有业务模块统一调用配置工具类，一处修改全系统生效
兜底默认值：配置缺失时自动使用内置默认值，不影响系统运行
变更留痕：所有配置修改写入操作日志，可追溯
1.4 功能全景（6 大分类，覆盖全系统）

三、后端完整实现（带缓存 + 全局工具类，全系统联动）
3.1 核心配置工具类 ConfigUtils.java
所有业务模块统一调用此工具类读取配置，实现全系统联动。优先读 Redis 缓存，缓存未命中查数据库，都没有返回默认值。
java
运行
@Component
@Slf4j
public class ConfigUtils {

    private static final String CACHE_PREFIX = "sys:config:";

    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    @Autowired
    private SysConfigMapper configMapper;

    // 静态注入
    private static ConfigUtils instance;

    @PostConstruct
    public void init() {
        instance = this;
    }

    /**
     * 获取字符串配置值
     */
    public static String getString(String key) {
        return getString(key, null);
    }

    /**
     * 获取字符串配置值（带默认值兜底）
     */
    public static String getString(String key, String defaultValue) {
        try {
            // 1. 先查缓存
            String cacheValue = instance.redisTemplate.opsForValue().get(CACHE_PREFIX + key);
            if (StrUtil.isNotBlank(cacheValue)) {
                return cacheValue;
            }
            // 2. 查数据库
            SysConfig config = instance.configMapper.selectOne(
                    Wrappers.<SysConfig>lambdaQuery().eq(SysConfig::getConfigKey, key)
            );
            if (config != null && StrUtil.isNotBlank(config.getConfigValue())) {
                // 写入缓存
                instance.redisTemplate.opsForValue().set(CACHE_PREFIX + key, config.getConfigValue());
                return config.getConfigValue();
            }
            // 3. 返回默认值
            return defaultValue != null ? defaultValue : (config != null ? config.getDefaultValue() : defaultValue);
        } catch (Exception e) {
            log.error("读取配置异常，key:{}", key, e);
            return defaultValue;
        }
    }

    /**
     * 获取数字型配置
     */
    public static Integer getInt(String key, Integer defaultValue) {
        String value = getString(key);
        if (StrUtil.isBlank(value)) return defaultValue;
        try {
            return Integer.parseInt(value);
        } catch (NumberFormatException e) {
            return defaultValue;
        }
    }

    /**
     * 获取布尔型配置
     */
    public static Boolean getBoolean(String key, Boolean defaultValue) {
        String value = getString(key);
        if (StrUtil.isBlank(value)) return defaultValue;
        return "true".equalsIgnoreCase(value);
    }

    /**
     * 刷新指定配置缓存
     */
    public static void refreshCache(String key) {
        instance.redisTemplate.delete(CACHE_PREFIX + key);
    }

    /**
     * 刷新全部配置缓存
     */
    public static void refreshAllCache() {
        Set<String> keys = instance.redisTemplate.keys(CACHE_PREFIX + "*");
        if (CollUtil.isNotEmpty(keys)) {
            instance.redisTemplate.delete(keys);
        }
    }
}

@Service
public class SysConfigService {

    @Autowired
    private SysConfigMapper configMapper;

    /**
     * 按分组查询配置列表
     */
    public List<SysConfig> listByGroup(String group) {
        return configMapper.selectList(
                Wrappers.<SysConfig>lambdaQuery()
                        .eq(StrUtil.isNotBlank(group), SysConfig::getConfigGroup, group)
                        .orderByAsc(SysConfig::getSort)
        );
    }

    /**
     * 批量更新配置
     */
    @Transactional(rollbackFor = Exception.class)
    @Log(module = "系统设置", type = "修改", desc = "修改系统基础配置")
    public void updateBatch(List<SysConfig> configList, Long operatorId) {
        for (SysConfig config : configList) {
            config.setUpdateBy(operatorId);
            configMapper.updateById(config);
            // 更新后刷新缓存
            ConfigUtils.refreshCache(config.getConfigKey());
        }
    }

    /**
     * 刷新全部缓存
     */
    public void refreshAll() {
        ConfigUtils.refreshAllCache();
    }
}

@RestController
@RequestMapping("/system/config")
@PreAuthorize("hasRole('admin')")
@Tag(name = "系统基础配置")
public class SysConfigController {

    @Autowired
    private SysConfigService configService;

    @GetMapping("/list")
    @Operation(summary = "按分组查询配置列表")
    public Result<List<SysConfig>> list(@RequestParam(required = false) String group) {
        return Result.success(configService.listByGroup(group));
    }

    @PutMapping("/batch")
    @Operation(summary = "批量更新配置")
    public Result<Void> updateBatch(@RequestBody List<SysConfig> configList) {
        Long userId = SecurityUtils.getCurrentUserId();
        configService.updateBatch(configList, userId);
        return Result.success();
    }

    @PostMapping("/refresh")
    @Operation(summary = "刷新全部配置缓存")
    public Result<Void> refresh() {
        configService.refreshAll();
        return Result.success();
    }
}

3.4 启动时初始化缓存 ConfigInitRunner.java
项目启动自动加载全部配置到 Redis，避免首次查询慢。
java
运行
@Component
@Slf4j
public class ConfigInitRunner implements ApplicationRunner {

    @Autowired
    private SysConfigMapper configMapper;
    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    private static final String CACHE_PREFIX = "sys:config:";

    @Override
    public void run(ApplicationArguments args) {
        log.info("开始加载系统配置到缓存...");
        List<SysConfig> allConfig = configMapper.selectList(Wrappers.emptyWrapper());
        for (SysConfig config : allConfig) {
            if (StrUtil.isNotBlank(config.getConfigValue())) {
                redisTemplate.opsForValue().set(CACHE_PREFIX + config.getConfigKey(), config.getConfigValue());
            }
        }
        log.info("系统配置缓存加载完成，共{}项", allConfig.size());
    }
}

const activeTab = ref('system')
const saving = ref(false)
const allConfig = ref([])

const formData = reactive({
  system: {},
  user: {},
  business: {},
  file: {},
  notice: {},
  backup: {}
})

// 加载配置
const loadConfig = async (group) => {
  const res = await getConfigList(group)
  const list = res.data
  // 按键名映射到对应分组
  list.forEach(item => {
    formData[group][item.configKey] = item
  })
  allConfig.value.push(...list)
}

// 保存配置
const saveConfig = async () => {
  saving.value = true
  try {
    // 收集当前分组的所有配置
    const currentGroup = activeTab.value
    const updateList = Object.values(formData[currentGroup])
    await updateConfigBatch(updateList)
    ElMessage.success('配置保存成功，已实时生效')
  } finally {
    saving.value = false
  }
}

// 重置
const resetConfig = () => {
  loadConfig(activeTab.value)
  ElMessage.info('已重置为当前保存值')
}

// 刷新缓存
const refreshCache = async () => {
  await refreshConfigCache()
  ElMessage.success('缓存刷新成功')
}

onMounted(() => {
  // 初始化加载全部分组
  ;['system', 'user', 'business', 'file', 'notice', 'backup'].forEach(group => {
    loadConfig(group)
  })
})
</script>

五、全系统联动机制说明
5.1 前端全局联动
系统名称：页面标题、登录页标题、首页顶部，均从配置读取
默认头像：成员档案无头像时，自动显示配置的默认头像
页脚版权：所有页面底部统一读取配置
5.2 后端业务联动
表格
模块	读取的配置项	作用
登录认证	user.passwordErrorLockCount、user.lockTime、user.sessionTimeout	密码锁定、会话超时控制
成员档案	user.defaultPassword、file.defaultAvatar	新建账号默认密码、默认头像
任务待办	business.taskExpireRemind	任务到期是否自动发提醒
工作周报	business.weeklyDeadline、business.weeklyRemind、notice.weeklyRemindTime	周报截止时间、提醒开关、提醒时间
文件上传	file.maxSize、file.allowTypes	全局上传校验，所有附件上传统一走此规则
操作日志	business.logKeepDays	定时清理过期日志的天数依据
数据备份	backup.keepCount	自动清理过期备份的份数依据
5.3 扩展新配置项的方式
无需改表结构、无需改工具类，两步完成：
数据库插入一条新配置（指定 key、分组、默认值）
业务代码中直接调用 ConfigUtils.getString("你的key") 读取
六、安全与可靠性保障
双层兜底机制：Redis 缓存 → 数据库 → 代码默认值，任意一层故障都不影响系统运行
权限严格控制：只有管理员能修改配置，普通用户无任何配置接口访问权限
变更留痕：所有配置修改自动写入操作日志，记录修改人、修改时间
格式校验：保存配置时校验数值范围、格式，防止非法值导致业务异常
缓存一致性：修改配置后立即删除缓存，下次读取自动加载最新值，保证实时生效

八、扩展建议
配置版本回滚：记录每次配置变更快照，支持一键回滚到上一版本
导入导出配置：支持配置导出为 JSON，迁移环境时一键导入
多环境配置：区分开发、测试、生产环境配置，自动切换
配置变更通知：关键配置（如备份策略、安全策略）修改后，自动通知所有管理员
